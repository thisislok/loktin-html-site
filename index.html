CI/CD Pipeline Project

Hosting this Website using Azure Blob Storage and GitHub Actions

This project demonstrates a full CI/CD pipeline where I build, test, and deploy this static website by pushing code from my local GitHub repository to Azure Blob Storage static website hosting.

Below is a step-by-step illustrated guide showing how I:

  • Create and configure an Azure Storage Account for static website hosting
  • Enable static website hosting on Azure Blob Storage
  • Upload website files to the special $web container
  • Set up a GitHub Actions workflow to automatically deploy changes on git push

Step 1: Deploy a Static Website

  1. Go to the Azure Portal. Click Create a resource > Storage > Storage account.
  2. Configure required settings (Subscription, Resource Group, unique Storage Account name, Region, Performance as Standard, Azure Blob Storage as Primary Service, LRS redundancy).
  3. Click Review + Create, then Create.

Step 2: Enable Static Website Hosting

  1. Open the storage account. Navigate to Static website (under Data Management), click Enable.
  2. Enter homepage file name (e.g., index.html) and error page (error.html).
  3. Click Save. Azure will generate a Primary Endpoint URL for your website.

Step 3: Upload Website Files

  1. Navigate to Storage Explorer or Containers in Azure Portal.
  2. Open the $web container.
  3. Click Upload and select your static files (HTML, CSS, JavaScript).
  4. Click Upload to publish.

Step 4: Build a CI/CD Pipeline with GitHub Actions

Using GitHub Actions, I automated deployments so that every git push to the main branch triggers this workflow:

  • Checkout the repo code
  • Run Azure CLI command to upload website files to Azure Blob Storage

Key GitHub Actions Workflow Code

name: Deploy to Azure Static Website

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Upload files to Azure Storage
        uses: azure/cli@v1
        with:
          inlineScript: |
            az storage blob upload-batch --account-name ${{ secrets.AZURE_STORAGE_ACCOUNT }} \
            --account-key ${{ secrets.AZURE_STORAGE_KEY }} \
            --destination '$web' \
            --source '.' \
            --overwrite
