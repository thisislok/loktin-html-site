  • Go to the Azure Portal > Create a resource > Storage > Storage account.

  • Click Save. Azure gives you a public website URL.

üîÑ Step 4: CI/CD Pipeline with GitHub Actions

This project is fully automated: I push updates to GitHub, and my website updates instantly on Azure using a CI/CD pipeline.

  • Created a GitHub repo and pushed my website code.
  • In Azure, generated a Deployment Token and stored it in GitHub Secrets as AZURE_STORAGE_KEY and AZURE_STORAGE_ACCOUNT.
  • Added a GitHub Actions workflow in .github/workflows/deploy.yml.

My workflow looks like this:

name: Deploy to Azure Static Website

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Upload files
        uses: azure/CLI@v1
        with:
          inlineScript: |
            az storage blob upload-batch --account-name ${{ secrets.AZURE_STORAGE_ACCOUNT }} \
            --account-key ${{ secrets.AZURE_STORAGE_KEY }} \
            --destination '$web' --source '.' --overwrite

Every time I push to main, this pipeline uploads the latest files to Azure!
